# AGENTS.md

## Codex Rules
1. Codex must adhere to the project's coding standards.
2. Code reviews are mandatory before merging any changes.
3. Maintain clear documentation for all functions and modules.
4. Loop in team members for feedback on major features.

## Developer Guidelines
1. Always work in a new branch for features or bug fixes.
2. Commit regularly with clear messages describing changes.
3. Keep the main branch deployable at all times.
4. Participate in regular team meetings to discuss progress and blockers.
5. Be respectful of others' opinions and time.

## Communication
- Use the project management tool for task assignments and tracking.
- Keep discussions on relevant channels to avoid clutter.
- Provide updates on your tasks in daily stand-ups.
